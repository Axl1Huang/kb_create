# 数据提取优先级分析

## 1. 优先级分类标准

### 高优先级（必须提取）
- 影响数据库插入成功与否的关键字段
- 建立表间关系必需的信息
- 论文核心元数据

### 中优先级（推荐提取）
- 增强数据价值和可用性的字段
- 支持高级查询和分析的字段
- 提升用户体验的信息

### 低优先级（可选提取）
- 补充性信息
- 可以通过外部数据源获取的信息
- 对核心功能影响较小的字段

## 2. 各类数据提取优先级

### 一级表数据（高优先级）
1. **研究领域 (research_field)**
   - id: 高优先级（必需）
   - field_name: 高优先级（必需）
   - field_code: 高优先级（必需）
   - category: 高优先级（必需）
   - description: 中优先级（推荐）

2. **期刊/会议 (venue)**
   - id: 高优先级（必需）
   - venue_name: 高优先级（必需）
   - venue_type: 高优先级（必需）
   - publisher: 中优先级（推荐）
   - impact_factor: 中优先级（推荐）

### 二级表数据（高优先级）
3. **关键词 (keyword)**
   - id: 高优先级（必需）
   - keyword_name: 高优先级（必需）
   - field_id: 高优先级（必需）
   - description: 低优先级（可选）

### 三级表数据（高优先级）
4. **文献 (paper)**
   - id: 高优先级（必需）
   - title: 高优先级（必需）
   - abstract: 高优先级（必需）
   - publication_year: 高优先级（必需）
   - venue_id: 高优先级（必需）
   - doi: 高优先级（必需）
   - language: 中优先级（推荐）

5. **作者 (author)**
   - id: 高优先级（必需）
   - author_name: 高优先级（必需）
   - affiliation: 中优先级（推荐）

### 关联表数据（高优先级）
6. **文献-关键词映射 (paper_keyword)**
   - paper_id: 高优先级（必需）
   - keyword_id: 高优先级（必需）
   - is_primary: 中优先级（推荐）
   - relevance_score: 中优先级（推荐）

7. **文献-作者关联 (paper_author)**
   - paper_id: 高优先级（必需）
   - author_id: 高优先级（必需）
   - author_order: 高优先级（必需）
   - is_corresponding: 中优先级（推荐）

### 扩展数据（中低优先级）
8. **文献元数据 (paper_metadata)**
   - paper_id: 高优先级（必需）
   - meta_key: 高优先级（必需）
   - meta_value: 高优先级（必需）
   - meta_type: 中优先级（推荐）

9. **实验数据**
   - HRT条件: 中优先级（推荐）
   - 污染物信息: 中优先级（推荐）
   - 酶活性数据: 中优先级（推荐）
   - 其他量化结果: 中优先级（推荐）

## 3. 提取难度和优先级平衡

### 易于提取且高优先级
1. 论文标题、摘要
2. 作者姓名和机构
3. 关键词列表
4. 发表年份、DOI
5. 期刊名称

### 难以提取但高优先级
1. 作者顺序和通讯作者标识（需要推理）
2. 关键词相关性评分（需要计算或估计）
3. 研究领域分类（需要专业知识判断）

### 易于提取但低优先级
1. 期刊影响因子（可查询）
2. 作者邮箱、ORCID（可能不存在于论文中）
3. 期刊ISSN（可查询）

## 4. 建议的提取策略

### 第一阶段（核心数据）
1. 提取论文基本信息（标题、摘要、年份、DOI）
2. 提取作者信息（姓名、机构）
3. 提取关键词信息
4. 提取期刊信息
5. 生成必需的ID

### 第二阶段（关联数据）
1. 确定作者顺序和通讯作者
2. 确定关键词与论文的主要关系
3. 计算相关性评分
4. 建立研究领域分类

### 第三阶段（扩展数据）
1. 提取实验数据
2. 补充元数据信息
3. 获取外部数据补充
